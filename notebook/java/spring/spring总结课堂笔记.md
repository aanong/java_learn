- spring循环依赖

  ---------------------------------------------------------------------------------------

  1.查缓存A ：没有

  2.A标记创建

  3.A实例化

  4.把A的工厂放入三级缓存

  5.属性注入B   (这个地方就是递归，创建完B后会回到这个位置)

  6.初始化A,后置通知生成代理A或者原生A,但是如果你曾经生成过，直接返回原生的A

  7.从二级缓存中拿出代理A或者原生A去替换

  8.删除A的创建标识

  9.把A放入到一级缓存，删除二级和三级缓存

  ---------------------------------------------------------------------------------------

​	1.查缓存B ：没有

​	2.B标记创建

​	3.B实例化

​	4.把B的工厂放入三级缓存

​	5.属性注入A,  代理A或者原生A放入二级缓存，删掉三级缓存中的工厂，并且会记录是否生成代理的标识

​	6.初始化B  

​	7.走后置通知生成代理B（取决于需不需要生成代理，可能是原生的B）

​	8.删除B的创建标识

​	9.把B放入到一级缓存，删除二级和三级缓存

- spring aop

  1.分为两个步骤

  ​	启动时：

  ​			1.获取到切面

  ​			2.给方法和切面对应的拦截通知绑定到一起（每个方法对应一个拦截链）

  ​			3.生成代理

  ​	调用时：索引加递归

  AOP什么样的情况下用jdk，什么样的情况用cglib?

  如果目标类是接口或者是代理类，则直接使用JDKproxy，有没有配置优先级，如果配置了优先级，那么默认使用cglib
